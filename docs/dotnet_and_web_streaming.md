## 一、 K230 本地网卡与“直接接线”连接方式（完全可以！）

### 1.1 K230 自带本地网卡吗？
**是的！K230 芯片原生集成了以太网 (Ethernet MAC) 控制器与 USB OTG 控制器。**
市面上绝大多数 K230 开发板（如勘智官方 K230 CanMV 板、立创 K230 开发板、01Studio K230 等）均原生支持以下两种**直接插线**的物理网卡连接方式：

#### 方式一：标准 RJ45 网线直接插网口（最稳定）
- **操作**：直接拿一条标准 RJ45 网线，一端插在 K230 开发板的 RJ45 网口上，另一端插在电脑或路由器上。
- **优点**：百兆/千兆带宽极高，延迟极低（< 5ms），不需要配置无线密码，即插即用。

#### 方式二：USB Type-C 数据线虚拟网卡直连（最方便，连网线都不需要！）
- **操作**：用普通 Type-C 数据线将 K230 的 USB 口插到电脑上。
- **原理**：K230 固件默认会通过 USB 协议模拟一个 **RNDIS / CDC-ECM 虚拟网卡**。
- **访问**：电脑插入 Type-C 线后，系统会自动识别出一个虚拟网卡（例如内网固定 IP `192.168.42.1` 或 `192.168.137.2`）。在电脑浏览器中直接输入 `http://192.168.42.1:8080` 就能立刻看画框图像！

## 二、 基于直接接线的 Web 实时画框图像串流

结合 K230 开发板的**网线接口**或 **Type-C 虚拟网卡**，可以将摄像头采集并由 KPU 推理画框后的画面，在 Web 端实时直播。

### 2.1 架构原理 (MJPEG HTTP Streaming)
```text
  ┌─────────────────┐       ┌─────────────────┐       ┌─────────────────┐
  │ K230 摄像头取帧  │  ───► │  KPU 运行 YOLO   │  ───► │ 图像绘制检测框   │
  │ (media.camera)  │       │ (nncase_runtime)│       │ (draw_rectangle)│
  └─────────────────┘       └─────────────────┘       └─────────────────┘
                                                               │
  ┌─────────────────┐       ┌─────────────────┐                │
  │ 电脑浏览器打开看  │ ◄───  │ HTTP Web Server │ ◄──────────────┘
  │ (http://IP:8080)│  (MJPEG)│ (socket/stream) │ (HW JPEG 压缩)
  └─────────────────┘       └─────────────────┘
```

1. **硬件 JPEG 编码**：使用 `img.compress(quality=75)` 将渲染好边框的帧快速压缩为 JPEG 二进制数据。
2. **HTTP 协议**：采用 HTTP 标准 `multipart/x-mixed-replace; boundary=frame` 协议头，无需任何插件，原生浏览器即可播放视频流。

### 2.2 示例代码与运行

项目内置了完整的 Web 串流模板：[templates/canmv_k230_web_streamer.py](file:///d:/Destop/k230-project/templates/canmv_k230_web_streamer.py)。

#### 上板操作步骤：
1. **直接接线**：用 Type-C 数据线连电脑，或者用标准 RJ45 网线插板卡网口。
2. 将 `templates/canmv_k230_web_streamer.py` 拷贝至 SD 卡 `sharefs` 并重命名为 `main.py`。
3. 板端启动后，控制台会输出网络识别结果：
   ```text
   [NET] ✅ USB Type-C 虚拟网卡已直连 (RNDIS)，IP: 192.168.42.1
   ==================================================
    🌐 K230 Web 实时 AI 视频流服务已就绪!
    👉 请在同一局域网电脑/手机浏览器访问: http://192.168.42.1:8080/
   ==================================================
   ```
4. 电脑浏览器直接打开 `http://192.168.42.1:8080/`，即可看到实时 30FPS 带有 AI 检测框的网页视频流！

## 三、 .NET 依赖机制剖析与“纯 Python 开发者”替代方案

### 3.1 为什么模型转换阶段需要 .NET？
- **底层架构**：嘉楠官方的 NPU 编译器 `nncase`（用于将 `.onnx` 编译为 `.kmodel`）底层图优化器与算子融合主要是由 **C# / .NET 7.0** 编写的。

### 3.2 上板运行阶段：100% 纯 Python！
在 **K230 开发板** 侧，开发环境是 **100% 纯 Python (MicroPython / CanMV)**！
- 板端不需要安装任何 .NET 运行时。
- 直接通过 `import nncase_runtime as nn` 即可用标准的 Python 语法调用 KPU 加载模型、推理与画框。

### 3.3 免手动安装 .NET 的“纯 Python”方案
- **官方 Docker 容器方案（最推荐）**：使用 `ghcr.io/kendryte/k230_sdk` 官方镜像，在容器内部运行 `python tools/to_kmodel.py`，宿主机零依赖、零污染。
- **云端在线平台**：使用 01Studio 或 CanMV 线上 AI 转换平台直接生成 `.kmodel`。
